# Development Log - 2026-04-23

## Stage 4: PDF Caption Detection MVP

Replaced the stage 3 fake figure/table list with real caption scanning based on
PDF text content.

## Completed

- The reader figure panel now scans the active PDF reader instead of rendering
  fixed demo entries.
- It accesses the Zotero reader's PDF.js document through the active reader
  instance.
- It prefers rendered text layers/page views and falls back to
  `pdfDocument.getPage(pageNumber)` when needed.
- It groups PDF text items into approximate text lines by y coordinate and
  records caption positions for more precise jumps.
- It detects captions that start a line with patterns such as:
  - `Figure 1`
  - `Fig. 1`
  - `Table 1`
  - `图1`
  - `表1`
  - `图十二`
- The detected entries reuse the existing panel card UI and click navigation.
- The refresh button now rescans the current PDF.
- Added panel states:
  - scanning
  - no captions detected
  - scan failed
- Added sequence checks and deduplication:
  - Removes subfigure entries such as `Figure 4A` when `Figure 4` exists.
  - Warns when figure/table sequences appear to have gaps.
  - Normalizes Chinese figure numbers such as `图 1 0` to `图10`.
- Scanning runs without forcing page jumps during reading.
- Navigation anchors to caption position:
  - Figure captions are aligned near the bottom of the viewport.
  - Table captions are aligned near the top of the viewport.
- Zotero 9 compatibility is declared in the manifest.

## Known Limitations

- Detection is caption-based only. It does not yet find figure image bounding
  boxes.
- It only detects captions that begin a PDF text line.
- PDF text extraction order can vary between publishers, so multi-column papers
  may need additional heuristics.
- Exact location jumps depend on the rendered PDF text layer. If the text layer
  is unavailable, navigation falls back to page-level positioning.
- Thumbnails and enlarged previews are not implemented yet.
